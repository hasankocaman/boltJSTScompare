Cypress Komutları

JS
JavaScript

describe('Cypress Commands', () => {
  it('should handle API requests', () => {
    cy.request('POST', '/api/login', {
      username: 'test@example.com',
      password: 'password'
    }).then((response) => {
      expect(response.status).to.eq(200)
      expect(response.body).to.have.property('token')
    })
  })
})

// Custom Commands
Cypress.Commands.add('loginViaUI', (username, password) => {

  cy.get('[data-cy=password]').type(password)
  cy.get('[data-cy=login-btn]').click()

Hata Yönetimi

JS
JavaScript

describe('Error Handling', () => {
  it('handles API errors', () => {
    cy.request({
      method: 'GET',
      url: '/api/nonexistent',
      failOnStatusCode: false
    }).then((response) => {
      if (response.status === 404) {
        cy.log('Expected 404 error occurred')
      }
    })
  })

  it('handles element not found', () => {
    cy.visit('/page')
    cy.get('.maybe-exists', {timeout: 1000})
      .should('exist')
      .catch(() => {
        cy.log('Element not found, continuing...')
      })
  })
})

Detaylı Karşılaştırma

JavaScript Avantajları

  • Hızlı başlangıç, kurulum gerektirmez
  • Esnek ve dinamik yapı
  • Geniş topluluk desteği
  • Daha az boilerplate kod
  • Runtime'da tip değişikliği mümkün

JavaScript Dezavantajları

  • Runtime hataları daha fazla
  • IDE desteği sınırlı
  • Büyük projelerde maintainability sorunu
  • Tip güvenliği yok
  • Refactoring zorluklarï

TypeScript Avantajları

  • Compile-time tip kontrolü
  • Mükemmel IDE desteği
  • Daha güvenli refactoring
  • İyi dokümantasyon desteği
  • Büyük projeler için ideal

TypeScript Dezavantajları

  • Öğrenme eğrisi var
  • Ek konfigürasyon gerekir
  • Daha fazla boilerplate kod
  • Compile süresi
  • Bazı kütüphaneler için tip tanımı gerekir

Performans Karşılaştırması

JavaScript: %40 - Runtime'da hata yakalama

TypeScript: %90 - Compile-time hata yakalama

Maintainability

JavaScript: %35 - Büyük projelerde zorlu

TypeScript: %85 - Mükemmel maintainability

Öneriler ve Sonuç

🎯 Ne Zaman JavaScript?

Küçük projeler, hızlı prototipleme, öğrenme amaçlı projeler ve başlangıç seviyesindeki ekipler için JavaScript ideal bir seçimdir.

🚀 Ne Zaman TypeScript?

Büyük ölçekli projeler, kurumsal uygulamalar, uzun vadeli bakım gerektiren projeler ve deneyimli ekipler için TypeScript önerilir.

💡 Pro İpuçları

  • ✅ TypeScript ile başlıyorsanız, yavaş yavaş strict modu açın
  • ✅ Custom Cypress komutlarınız için mutlaka tip tanımları yazın
  • ✅ Fixture dosyalarınız için interface tanımları kullanın
  • ✅ Page Object Pattern ile testlerinizi organize edin
  • ✅ ESLint ve Prettier kullanarak kod kalitesini artırın
